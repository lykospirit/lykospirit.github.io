Motion Editing II
Several thoughts on the respective papers discussed this session:
  1. SketchiMo - Sketch-based Motion Editing for Articulated Characters (Choi et. al., 2016)
    Interesting, useful for quick storyboarding, easier to set up scenes and interactions between characters to get a high-level idea of how a scene should play out. When sketching the 2D line, how does the program decide what 3D line that maps to, since there are infinitely many possibilities?

  2. The Cartoon Animation Filter (Wang et. al., 2006)
    Takes a lot of ideas from computer vision and applies it to animation. Discretization of Laplacian of Gaussian is also used in CV to compute image gradients for, as an example, edge detection. The resulting animation still looks very janky and unnatural, however, and would probably need touching up by an animator.

  3. Motion Amplifiers - Sketching Dynamic Illustrations Using the Principles of 2D Animation (Kazi et. al., 2016)
    Much like (2), not convinced it would translate well to a 3D situation. The authors of this paper leaned heavily into animating specifically 2D sketches, however, and their technique works well with this medium.

  4. Neural State Machine for Character-Scene Interactions (Starke et. al., 2019)
    The results are very cool. Surprised that the results look that good given just 2 hours' worth of MoCap data. An interesting point on reliability was raised: is it possible to introduce sufficient constraints to guarantee realistic enough motion from the network?

Motion Editing I
While I had learned before that singularities exist when using Euler angles, it was good to see a concrete example of gimbal lock when working with Euler angles. The discussion on quaternions was also one of the most comprehensive and understandable introductions to the subject. It was also interesting to learn about the exponential map, and it would be interesting to see how the singularities in this representation manifest in an animation.

Motion graphs were also a very clever idea. An immediate application that stood out to me was in games, where the player can control a character through different interactions with the environment. While it may not work with something that requires extremely quick transitions, like in a sports game, it would work for, for example, transitioning between walking, running, climbing stairs, picking up objects, and so on.

A game that might (?) use a variant of this technique is Assassin's Creed, where the player regularly transitions between walking, running, fighting, and different climbing animations while navigating the environment.

Inverse Kinematics II
While the basic idea is shared between all the paper presented in this session, each brings to the table their own improvement to the technique, making IK work faster or more generally.

I am curious to see how these techniques fare against traditioanl Jacobian techniques, and how they perform in a professional setting both in terms of speed as well as the visual result. Bringing forward a question from the last session: do animators have a choice of several techniques, and do they prefer certain results over others?

Inverse Kinematics I
It is interesting to see that inverse kinematics uses similar techniques to solving other non-linear systems in other applications; for instance, image homologies and 3D reconstruction. The lecture brought up several questions:
  • Does modern software provide multiple options for solvers, given that each potentially provides different behaviors? Do animators have a "favourite" numerical solver that they are comfortable working with, or are the differences negligible to the human eye?
  • How do we prevent "jerky" motion that could result from the iterative nature of these algorithms, say, by picking a Δθ that is too large or inconsistent solving times in a real-time application?

MoCap Demo
I had never been in a MoCap studio before this semester and it was exciting to be able to see it in action. I was impressed by how quickly the input from multiple cameras, which were not necessarily at the same positions during each calibration and were aimed at a noisy and cluttered background, was parsed and analysed quickly enough to provide a realtime rendering of a skeletal pose. The examples of animations and research that was being done at the studio were also enlightening and got me thinking about the broad range of applications that motion captured data could be useful for.

Introduction to Technical Animation
Talking about the different forms of animation was insightful to understanding what to expect in the coming semester, and to seeing how animation has evolved across history. While there are numerous up and coming technologies that change how animations are created and rendered, it was interesting to see that understanding key, basic techniques like keyframing is still important for good animations.

The section on creating autonomous animation was also interesting because although the results were not as seamless and natural as manual keyframing could accomplish, their relative popularity in research points to the diverse applications of this technology, in interactive media for instance. It was interesting to think about the potential uses for these techniques while looking at the examples of physics-based and autonomous/learned animation.
